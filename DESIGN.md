# OpenList Sorter 设计说明

## 一句话目标

OpenList Sorter 是一个连接 OpenList/Alist 文件站的 Windows 桌面端远程文件分类工具。它用“连续预览 + 多选 tag + 批量提交”的方式，替代网页端逐个移动、复制文件的繁琐操作。

第一版重点是把功能流程跑通，界面保持现代化、清爽、可继续调整。

## 要解决的问题

用户之前把大量图片、视频或其他文件混在同一个 OpenList 目录里。OpenList 网页端可以移动、复制和预览文件，但面对大量文件时，逐个点开、选择目标目录、执行操作非常慢。

本工具借鉴机器学习标注系统的工作流，但第一版不做自动识别分类，而是做人工快速打标：

- 一个文件一个文件地看。
- 给当前文件选择一个或多个 tag。
- tag 不立即提交到服务端，只先保存在本地会话状态里。
- 点击保存后，统一调用 OpenList API 批量移动或复制。

## 基本用户流程

1. 打开程序。
2. 首先看到连接页面，类似登录页。
3. 输入 OpenList endpoint。
4. 在 token key 登录和账号密码登录之间二选一。
5. 根据需要勾选记住我和自动登录。
6. 连接成功后进入分类页面。
7. 使用远程目录浏览器选择待分类源目录。
8. 配置多个 tag，每个 tag 绑定一个目标目录。
9. 进入分类模式。
10. 程序只加载当前源目录下的一层文件，不递归扫描子目录。
11. 按 OpenList API 返回的原始顺序逐个显示文件。
12. 用户为当前文件选择一个或多个 tag。
13. 使用上一个、下一个连续处理。
14. 顶部实时显示已提交、已选择未提交、未处理、当前序号等状态。
15. 点击保存后，统一提交所有已选择但未提交的文件操作。

## 窗口与页面拆分

程序不把所有功能堆在一个页面。

启动连接页：

- 程序刚启动时显示。
- 包含 endpoint。
- 支持 token key 登录。
- 支持账号密码登录。
- 两种登录方式通过单选项二选一。
- 包含记住我、自动登录。
- 自动登录依赖记住我；勾选自动登录会自动勾选记住我，取消记住我会取消自动登录。

分类主页面：

- 连接成功后才显示。
- 不再展示 endpoint、token、账号密码等连接表单。
- 顶部是标准菜单栏。
- 中间是分类工作区。
- 底部状态栏显示当前 endpoint 和操作状态。

菜单栏：

- 文件：连接信息、重新连接、退出。
- 编辑：清空当前文件选择、清空全部未提交选择。
- 帮助：关于 OpenList Sorter。

连接信息窗口：

- 从菜单栏打开。
- 作为独立窗口展示当前 endpoint 和连接状态。
- 可从这里触发重新连接。

本地缓存：

- 使用 Qt `QSettings` 记录轻量配置。
- 记住我开启时保存 endpoint、登录方式和凭据。
- 自动登录开启时，启动连接页会自动尝试连接。
- 关闭记住我时清除保存的连接凭据。
- tag 配置和源目录也使用本地设置保存。

## 目录选择器

目录选择器应该像网页端或 Windows 文件管理器一样工作：

- 初始显示根目录或当前目录。
- 点击文件夹后再请求下一层内容。
- 不一次性递归加载整个文件站。
- 顶部显示当前路径。
- 支持返回上一级。
- 支持刷新当前目录。
- 支持选择“当前目录”。

目录选择器在两个场景复用：

- 选择待分类源目录。
- 为 tag 选择目标目录。

## 分类模式约束

进入分类模式后，源目录锁定：

- 进入前可以切换源目录。
- 进入后不能再切换源目录。
- 如果要换目录，需要重新开始一次分类任务。

这样可以避免当前文件列表、本地 tag 状态和远程目录不一致。

## 文件显示规则

程序每次聚焦一个当前文件。

图片文件：

- 直接加载图片预览。
- 尽量覆盖常见和较少见格式，如 jpg、jpeg、png、gif、bmp、webp、tif、tiff、heic、heif、avif、jxl、ico、svg，以及 cr2、cr3、nef、arw、dng、orf、rw2、raf、pef 等 RAW 后缀。
- 预览优先使用 Qt5 本地图片解码能力，包括随 Qt 部署的 imageformats 插件。
- Qt 无法解码时，程序尝试调用本机 ImageMagick `magick` 将临时图片转为 PNG 再显示，用于尽量支持 heic、heif、avif、raw 等格式。
- 如果本机没有 ImageMagick 或转码失败，程序退回文件信息显示，不影响打 tag 和提交。

视频文件：

- 使用视频控件加载媒体。
- 不自动播放。
- 用户手动点击播放/暂停。
- 第一版只面向常见视频格式，如 mp4、mkv、mov、avi、webm、flv、wmv、m4v、ts。

其他文件：

- 不强制预览。
- 显示文件名、大小、类型、修改时间、完整路径等信息。

界面始终显示当前文件所在路径，例如：

```text
/uploads/待分类/IMG_0012.jpg
```

## tag 规则

tag 由用户配置，每个 tag 映射到一个 OpenList 目标目录。

示例：

```text
风景 -> /照片/已分类/风景
人物 -> /照片/已分类/人物
表情包 -> /照片/已分类/表情包
资料 -> /资料/已分类
```

一个文件可以处于以下状态：

- 未选择 tag：未处理。
- 选择一个 tag：保存时移动到该 tag 对应目录。
- 选择多个 tag：保存时复制到多个 tag 对应目录。

当一个文件选择多个 tag 时，界面需要提示用户：保存时会进行复制。

第一版规则：

- 单 tag：move。
- 多 tag：copy 到多个目录，原文件保留在源目录。

后续可增加选项：

- 多 tag 复制成功后删除原文件。
- 多 tag 复制成功后移动到已处理目录。
- 覆盖同名文件或跳过同名文件的策略。

## 本地暂存机制

选择 tag 后，不立即调用 OpenList API。

程序维护一个本地会话状态：

- 当前文件列表。
- 当前文件索引。
- 每个文件已选 tag。
- 每个文件是否已提交。
- 每个文件提交失败信息。

只有点击保存时，才提交“已选择 tag 且尚未提交”的文件。

好处：

- 减少 API 请求次数。
- 避免误点后马上影响远程文件。
- 用户可以回到上一个文件修改 tag。
- 适合连续处理大量文件。

## 顶部统计

顶部显示当前任务状态：

```text
总文件数：300
已提交：80
已选择未提交：25
未处理：195
当前：106 / 300
```

含义：

- 已提交：已经成功调用移动或复制 API 的文件。
- 已选择未提交：已经选择 tag，但还没有保存。
- 未处理：没有选择 tag 且未提交。
- 当前：当前正在查看的文件序号。

## 保存提交规则

点击保存后，程序遍历已选择未提交的文件。

单 tag 文件：

```text
POST /api/fs/move
src_dir: 源目录
dst_dir: tag 目标目录
names: [文件名]
```

多 tag 文件：

```text
POST /api/fs/copy
src_dir: 源目录
dst_dir: tag A 目标目录
names: [文件名]

POST /api/fs/copy
src_dir: 源目录
dst_dir: tag B 目标目录
names: [文件名]
```

任意提交失败时，文件保持待提交状态，并在界面中显示错误信息。

## OpenList API

第一版使用以下 OpenList/Alist v3/v4 风格接口：

- `POST /api/auth/login`
- `POST /api/fs/list`
- `POST /api/fs/get`
- `POST /api/fs/move`
- `POST /api/fs/copy`

认证 header 使用：

```text
Authorization: <token>
```

不是 Bearer token。
